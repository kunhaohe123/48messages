# 口袋48房间消息抓取方案

## 一、抓包工具准备

### 1.1 推荐工具
- **Charles** (推荐) - 跨平台，界面友好
- **Fiddler** - Windows平台，功能强大
- **mitmproxy** - 命令行工具，支持脚本扩展

### 1.2 安装证书（以Charles为例）

#### PC端证书安装
1. 打开Charles → Help → SSL Proxying → Install Charles Root Certificate
2. 安装证书并设置为信任

#### 手机端证书安装
1. 确保手机和电脑在同一WiFi网络
2. 查看电脑IP：Help → Local IP Address
3. 手机WiFi设置代理：
   - 服务器：电脑IP
   - 端口：8888
4. 手机浏览器访问：`chls.pro/ssl`
5. 下载并安装证书
6. iOS需要信任证书：设置 → 通用 → 关于本机 → 证书信任设置

#### HTTPS配置
1. Charles → Proxy → SSL Proxying Settings
2. 启用SSL代理
3. 添加 Location：host: `*`，port: `443`

## 二、关键API接口分析

### 2.1 房间消息相关接口

根据开源项目分析，可能的接口模式：

```
# 获取房间消息
GET /api/room/{roomId}/messages
GET /api/v1/room/chat/list

# 发送消息
POST /api/room/{roomId}/message

# 获取成员信息
GET /api/member/{memberId}/info
GET /api/v1/user/profile

# 登录认证
POST /api/login
POST /api/v1/auth/login
```

### 2.2 关键请求头
```
User-Agent: 口袋48 App版本信息
X-Device-ID: 设备标识
X-Session-ID: 会话标识
Authorization: Token认证
Cookie: 登录Cookie
sign: 签名参数（可能需要）
```

### 2.3 认证参数
```
username: 手机号/账号
password: 密码
imei: 设备序列号
version: App版本（如 8.5.0）
timestamp: 时间戳
nonce: 随机数
```

## 三、抓包步骤详解

### 3.1 登录流程抓包
1. 清空Charles历史记录
2. 打开口袋48，点击登录
3. 输入手机号和验证码
4. 观察POST请求，记录：
   - 请求URL和参数
   - 请求头信息
   - 响应返回的Token/Session
   - 加密方式（是否有sign参数）

### 3.2 房间消息抓包
1. 登录后进入某个成员的房间
2. 等待房间消息加载
3. 分析HTTP请求：
   - URL结构
   - Query参数或Body参数
   - 响应数据格式（JSON）
4. 记录WebSocket连接（如果有实时消息）

### 3.3 需要关注的内容

#### 请求特征
- [ ] 请求URL完整路径
- [ ] HTTP方法（GET/POST）
- [ ] 请求参数格式（JSON/FormData/QueryString）
- [ ] 请求头中的认证信息
- [ ] 是否需要签名（sign）

#### 响应特征
- [ ] 数据格式（JSON/XML）
- [ ] 消息结构（userId, content, timestamp等）
- [ ] 分页机制（limit, offset, cursor等）
- [ ] 错误码定义

### 3.4 加密参数分析

如果发现sign签名：
1. 尝试禁用JavaScript（Web端）
2. 使用mitmdump脚本拦截原始数据
3. 分析请求参数组合规律
4. 常见签名方式：
   - MD5/HMAC参数拼接
   - Base64编码
   - 时间戳校验

## 四、模拟器抓包（推荐）

### 4.1 使用夜神/雷电模拟器
1. 安装口袋48 APK到模拟器
2. 设置Charles代理
3. 模拟器优势：
   - 可以Root查看加密算法
   - 设备信息固定
   - 易于重放测试

### 4.2 获取IMEI
模拟器中：
```bash
# 方法1：设置 → 关于平板 → 序列号
# 方法2：拨号界面输入 *#06#
# 方法3：使用RE文件管理器查看
/data/data/com.android.providers.telephony/databases/telephony.db
```

## 五、注意事项

### 5.1 法律合规
- 仅供个人学习研究使用
- 不要大规模爬取或商业使用
- 遵守口袋48用户协议
- 合理控制请求频率（建议≥60秒间隔）

### 5.2 技术建议
- 使用小号进行测试
- 记录所有请求参数作为参考
- 保存完整的抓包文件（.chlsj格式）
- 重点关注sign参数生成逻辑

### 5.3 常见问题
- **无响应**：检查代理设置和证书信任
- **登录失败**：可能需要更新App版本
- **签名错误**：分析加密算法或使用模拟器Root查看
- **IP限制**：降低请求频率或使用代理
