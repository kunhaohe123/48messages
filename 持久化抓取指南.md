# 口袋48持久化抓取方案

## 核心问题：能否不依赖手机？

### ✅ **答案是：可以！**

抓包只是用来**发现API接口**，就像破解密码一样——你只需要知道密码是什么，之后就可以直接用密码开门，不需要每次都去观察别人怎么开门。

## 持久化的关键技术

### 1. **Token认证机制**
```
登录流程：
1. 第一次：使用用户名/密码/IMEI登录 → 获取Token
2. 之后：使用Token调用API → 无需用户名密码
3. Token过期前：自动刷新或重新登录
```

### 2. **持久化运行需要的信息**

从抓包中获取以下内容就足够了：

| 信息 | 来源 | 用途 |
|------|------|------|
| API URL | 请求URL | 调用接口 |
| 请求参数 | Body/Query | 构造请求 |
| Token | 响应数据 | 身份认证 |
| IMEI | 抓包请求头 | 设备标识 |
| User-Agent | 请求头 | 模拟App |

### 3. **我已创建的持久化方案**

新增了 `persistent_scraper.py`，实现：

- **Token自动管理**
  - 保存Token到文件（`token.json`）
  - 自动检测Token有效期
  - 过期前自动刷新
  - 刷新失败自动重新登录

- **多房间监控**
  - 支持同时监控多个成员房间
  - 多线程独立抓取
  - 独立运行，互不影响

- **数据持久化**
  - SQLite数据库存储
  - 记录抓取日志
  - 统计数据概览

- **容错机制**
  - 连续失败自动重连
  - 异常处理不崩溃
  - 优雅停止

## 使用方法

### 1. 抓包获取关键信息（一次性）

使用Charles/Fiddler抓取：
- 登录接口和参数
- 房间消息接口
- Token格式
- IMEI值

### 2. 配置项目

```bash
cp config.example.json config.json
# 编辑 config.json 填入抓包获取的信息
```

### 3. 启动持久化抓取

```bash
# 首次运行（需要手机获取Token）
python persistent_scraper.py -c config.json

# 之后运行（自动使用保存的Token）
python persistent_scraper.py -c config.json

# 查看统计
python persistent_scraper.py -c config.json --stats
```

### 4. 后台运行（可选）

```bash
# 使用nohup后台运行
nohup python persistent_scraper.py -c config.json > output.log 2>&1 &

# 查看运行状态
ps aux | grep persistent_scraper

# 查看日志
tail -f output.log
```

## ⚠️ 注意事项

### Token有效期
- 不同平台Token有效期不同（1小时到30天）
- 需要定期检查是否过期
- 我的代码会自动处理

### 网络稳定性
- 建议在服务器上运行（24小时在线）
- 本地电脑运行需要保持开机

### 请求频率
- 建议间隔 ≥60秒
- 太频繁可能被封IP
- 我的代码默认60秒间隔

### 账号安全
- **强烈建议使用小号**
- 不要在主账号上测试
- 避免被封号风险

- API接口变化（需要重新分析）
